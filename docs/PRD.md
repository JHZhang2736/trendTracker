# 产品需求文档（PRD）
# TrendTracker — 全网趋势聚合与AI商业洞察平台

**文档版本**: v0.1
**创建日期**: 2026-03-19
**阶段**: MVP（个人内部使用）
**状态**: 草稿

---

## 1. 产品概述

### 1.1 产品背景

当前互联网趋势数据高度碎片化，分散在谷歌趋势、TikTok、百度指数、阿里指数等多个独立平台。商业决策者需要在多个平台间手动收集、对比、分析数据，耗时且容易遗漏信号。此外，原始趋势数据本身不能直接指导商业行动，需要结合具体业务场景进行解读。

### 1.2 产品定位

TrendTracker 是一款**全网趋势聚合 + AI商业洞察**工具。它将分散在各大平台的热门关键词、趋势数据统一采集、标准化处理，以可视化方式呈现，并通过大语言模型对趋势进行商业价值解读，辅助用户制定商业计划。

### 1.3 MVP阶段定位

- **使用对象**: 个人内部使用（产品所有者本人）
- **部署方式**: 本地运行，通过 Docker Compose 一键启动
- **核心目标**: 验证数据聚合可行性 + 验证AI商业洞察的实用价值

## 3. 问题陈述

### 3.1 用户当前面临的问题

1. **数据碎片化**: 趋势数据分散在5+个平台，每次分析需逐一手动查询
2. **无法跨平台对比**: 各平台数据格式不一致，难以判断哪个趋势真正值得关注
3. **数据到决策的断层**: 看到"某词搜索量上升"，不知道这意味着什么商业机会
4. **时效性差**: 手动查询耗时，可能错过趋势窗口期
5. **商业视角缺失**: 现有工具只展示数据，不提供"我该怎么做"的建议

### 3.2 产品解决的核心问题

> **将多平台原始趋势信号，转化为"今天我可以采取什么商业行动"的具体建议。**

## 4. 产品目标

| 目标 | 衡量标准 |
|------|----------|
| 数据聚合 | 每日自动采集各平台热门关键词，无需手动操作 |
| 统一展示 | 在一个界面看到所有平台趋势，支持排序和筛选 |
| AI洞察 | 每个趋势配套AI商业建议，覆盖电商、投资、内容等多个维度 |
| 告警通知 | 重要趋势出现时主动推送，而非被动查询 |
| 易于使用 | Docker Compose 一键启动，无需复杂配置 |

## 5. 功能范围（Scope）

### 5.1 MVP 包含（In Scope）

#### F1 — 数据采集模块

| 功能ID | 功能名称 | 描述 |
|--------|----------|------|
| F1.1 | 谷歌趋势采集 | 采集 Google Trends 热搜词、趋势指数，支持按地区和时间段查询 |
| F1.2 | 百度指数采集 | 采集百度指数热搜词及指数变化 |
| F1.3 | 阿里指数采集 | 采集阿里/淘宝平台热门搜索词和品类趋势 |
| F1.4 | TikTok趋势采集 | 采集 TikTok Creative Center 公开趋势数据（热门话题、热门商品） |
| F1.5 | 微博热搜采集 | 采集微博实时热搜榜单 |
| F1.6 | 定时自动采集 | 支持设置采集频率（如每天早6点自动采集一次），无需手动触发 |
| F1.7 | 历史数据存储 | 将采集结果持久化存储，支持查看历史趋势数据 |

#### F2 — 数据展示模块

| 功能ID | 功能名称 | 描述 |
|--------|----------|------|
| F2.1 | 趋势总览仪表盘 | 首页展示各平台当日热词，按平台分区展示 |
| F2.2 | 趋势热力图 | 以热力图形式展示关键词热度强度，支持多平台对比 |
| F2.3 | 趋势折线图 | 单个关键词的热度随时间变化曲线 |
| F2.4 | 跨平台收敛评分 | 当同一关键词在多个平台同时上升时，计算并展示"收敛评分"（越高越可能爆款） |
| F2.5 | 关键词搜索 | 支持搜索特定关键词，查看其在各平台的热度表现 |
| F2.6 | 分类筛选 | 支持按品类筛选趋势（如：消费品、科技、娱乐、金融等） |
| F2.7 | 趋势生命周期标签 | 自动标注趋势所处阶段：萌芽 / 上升 / 顶峰 / 衰退 |
| F2.8 | 快捷跳转链接 | 在趋势详情页提供一键跳转入口：1688搜同款、淘宝/天猫同品类、亚马逊同类目、相关股票行情页，减少从洞察到行动的操作步骤 |

#### F3 — AI商业洞察模块

| 功能ID | 功能名称 | 描述 |
|--------|----------|------|
| F3.1 | 趋势商业价值分析 | 对单个趋势进行AI分析，输出该趋势的商业机会解读 |
| F3.2 | 多维度机会建议 | AI从多个商业维度给出建议，包括但不限于：电商选品、内容创作选题、投资方向、创业方向、本地服务机会 |
| F3.3 | 每日商业简报 | 每天自动生成当日趋势摘要报告，包含Top趋势和对应AI机会建议 |
| F3.4 | 批量分析 | 支持选择多个趋势词，批量发送给AI进行综合分析 |
| F3.5 | 自由对话 | 支持用户基于当前趋势数据，与AI进行自由问答（如"根据今天的趋势，我应该重点关注哪个方向？"） |
| F3.6 | 分析历史记录 | 保存每次AI分析结果，支持回顾历史洞察 |
| F3.7 | 情感极性标签 | AI自动判断趋势的情感倾向：正面 / 中性 / 负面，防止将危机事件（召回、负面舆情）误判为商业机会 |
| F3.8 | 相关词扩展 | 对任意热词，AI自动推荐5-10个语义相关的延伸词，帮助用户快速扩大监控范围（如"露营"→"天幕/折叠椅/户外炊具"） |

#### F4 — 告警与通知模块

| 功能ID | 功能名称 | 描述 |
|--------|----------|------|
| F4.1 | 关键词监控 | 用户可添加关键词到监控列表，当该词热度变化超过阈值时触发告警 |
| F4.2 | 收敛评分告警 | 当某关键词收敛评分超过设定阈值时触发告警 |
| F4.3 | 邮件通知 | 支持发送告警通知到邮件 |
| F4.4 | 每日简报推送 | 每天定时推送当日趋势简报到邮件 |

#### F5 — 系统基础功能

| 功能ID | 功能名称 | 描述 |
|--------|----------|------|
| F5.1 | 本地部署 | 支持 Docker Compose 一键启动所有服务 |
| F5.2 | 配置管理 | 通过配置文件管理 AI模型API Key、采集频率、告警阈值等参数 |
| F5.3 | 数据管理 | 支持查看存储使用量，支持清理过期数据 |

### 5.2 MVP 不包含（Out of Scope）

- 用户注册 / 登录 / 多用户体系
- 移动端 App
- 公开对外的 SaaS 化
- 付费订阅功能
- 社交分享功能
- Amazon / Shopify 等电商平台数据接入
- 股票实时行情数据接入（AI可基于趋势给出方向建议，但不接入K线/报价数据）
- 自动执行商业操作（如自动上架商品、自动下单股票）

## 6. 用户故事（核心场景）

### 场景一：早间趋势巡检
> 早上打开 TrendTracker，看到今日趋势仪表盘，发现"某款户外产品"在谷歌趋势和阿里指数同时上升，收敛评分达到85分。AI建议："该品类正处于春季备货窗口期，建议考虑在1688批量采购并上架速卖通或亚马逊，参考近期爆款链接风格进行主图设计。"用户将该条目添加到"待研究"列表。

### 场景二：宏观事件洞察
> 某天百度指数出现"美联储加息"搜索量激增，TikTok同步出现相关话题。AI分析："本次加息预期升温，历史上利好黄金和美元资产，建议关注黄金ETF和美元理财产品，同时消费电子等周期性行业可能承压。"用户据此调整个人资产配置思路。

### 场景三：内容选题
> 用户是兼职内容创作者。周末查看趋势发现"AI学习工具"相关词在TikTok和微博同步热度上升。AI建议："该话题适合制作'XX款AI工具测评'或'用AI提升工作效率'系列内容，受众主要为职场新人，建议以短视频+图文形式发布。"

### 场景四：自定义告警
> 用户在监控列表添加"露营"相关词，设置阈值为"任意平台单日涨幅>50%"。某周五下午系统发送邮件告警，用户在下班前看到通知，立即在1688上查看露营用品供货情况，抓住了周末爆单窗口。

## 7. 非功能性需求

| 类别 | 要求 |
|------|------|
| 性能 | 仪表盘首屏加载 < 3秒；AI分析响应 < 30秒 |
| 可用性 | 本地运行，无需互联网（除数据采集和AI调用时） |
| 数据保留 | 默认保留90天历史趋势数据 |
| 采集稳定性 | 单次采集失败自动重试，不影响整体服务 |
| 配置友好 | 所有关键参数可通过配置文件修改，无需改代码 |
| 资源占用 | 在普通个人电脑（8GB内存）上可流畅运行 |

| 验证目标 | 成功标准 |
